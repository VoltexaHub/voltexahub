# v0.8.0 Build Plan — REVISED
**Target release:** April 5, 2026
**Theme:** Real Theme System — Component Overrides + Full Customization

---

## Overview
Build a theme system where community members can download stock themes, modify Vue components, add/remove sections, change layouts — then upload their custom theme. Like MyBB/XenForo themes but for a Vue 3 SPA.

---

## Architecture

### How Themes Work
```
my-theme/
├── theme.json              # manifest
├── theme.css               # color/style overrides (CSS variables)
├── preview.png             # screenshot
└── components/             # Vue component overrides (compiled JS)
    ├── ThreadRow.vue        # custom thread list row
    ├── ProfileHeader.vue    # custom profile header
    └── HomeHero.vue         # custom home page hero section
```

### Theme Resolution Order
1. Check active theme for component override → use it
2. Fall back to default component

### Compile Pipeline
1. Theme author writes .vue files (using our SDK/starter)
2. They run `npm run build` locally (we provide a theme dev kit with Vite config)
3. Build outputs compiled JS modules per component
4. Theme zip contains compiled JS (not raw .vue source)
5. Admin uploads zip → backend stores it → frontend loads overrides dynamically

Each overridable component:

- Uses composables for data (not inline API calls)
- Exports a stable interface theme authors can rely on
- Wrapped in a `<ThemeComponent name="ThreadRow" :props="..."/>` resolver

### Dynamic Component Resolver (Frontend)
Create `src/theme/resolver.js`:
```js
// Registry of overridable components
const defaults = {
  'ThreadRow': () => import('../components/forum/ThreadRow.vue'),
  'PostCard': () => import('../components/forum/PostCard.vue'),
  'ForumCard': () => import('../components/forum/ForumCard.vue'),
  // ... etc
}

// Theme overrides loaded from active theme
const overrides = {}

export function resolveComponent(name) {
  if (overrides[name]) return overrides[name]
  return defaults[name]
}

export function registerOverride(name, loader) {
  overrides[name] = loader
}
```

Create `src/components/ThemeComponent.vue`:
```vue
<script setup>
import { resolveComponent } from '../theme/resolver'
const props = defineProps({ name: String, ...componentProps })
const component = resolveComponent(props.name)
</script>
<template>
  <component :is="component" v-bind="$attrs" />
</template>
```

### Theme Loader (Frontend)
On app boot, if active theme has component overrides:
1. Fetch theme manifest from backend
2. Dynamically import compiled JS modules from `/api/themes/{slug}/component/{name}.js`
3. Register each as an override in the resolver
4. Components swap in automatically

## Phase 2: Theme Backend + Compile (Week 1-2)

### Backend Theme Endpoints (extend existing)
- `GET /api/themes/{slug}/manifest` — return theme.json with list of overridden components
- `GET /api/themes/{slug}/component/{name}.js` — serve compiled component JS
- Theme upload now accepts compiled JS files in components/ directory
- Validate uploaded components against allowed override names

### Theme Storage
```
storage/app/themes/{slug}/
├── theme.json
├── theme.css
├── preview.png
└── components/
    ├── ThreadRow.js     # compiled Vue component
    ├── PostCard.js
    └── ...
```

## Phase 4: Theme Dev Kit

### voltexahub-theme-sdk (npm package / GitHub repo)
- Vite config pre-configured for building theme components
- TypeScript types for all component props
- All composables (useAuth, useForumStore, etc.) available
- Example theme to fork
- `npm run dev` — preview your theme with hot reload against a VoltexaHub instance
- `npm run build` — compile .vue overrides to JS modules
- `npm run package` — create uploadable theme.zip

### Documentation (docs.voltexahub.com)
- Theme authoring guide
- Component reference (every overridable component, its props, slots)
- CSS variable reference
- Tutorial: "Create your first theme"
- Tutorial: "Override a component"
