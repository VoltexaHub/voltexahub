**Theme:** Community Themes — Full System

## Overview
Give communities full visual customization without touching Vue source code. CSS custom properties are the unlock — one file changes everything.

## Week 1: Foundation

### CSS Variables Refactor (Frontend)
- Audit every hardcoded color in the codebase (`bg-[#0a0a0f]`, `text-[#7c3aed]`, etc.)
- Define complete CSS variable set in `:root` block:
  ```css
  :root {
    --vh-bg-base: #0a0a0f;
    --vh-bg-card: #0f0f1a;
    --vh-bg-elevated: #13131f;
    --vh-bg-input: #0f0f1a;
    --vh-border: #1e1e30;
    --vh-border-subtle: #161622;
    --vh-accent: #7c3aed;
    --vh-accent-hover: #6d28d9;
    --vh-accent-soft: rgba(124,58,237,0.1);
    --vh-accent-light: #a78bfa;
    --vh-text-primary: #f1f5f9;
    --vh-text-secondary: #e2e8f0;
    --vh-text-muted: #94a3b8;
    --vh-text-faint: #475569;
    --vh-success: #10b981;
    --vh-warning: #f59e0b;
    --vh-danger: #ef4444;
    --vh-info: #3b82f6;
    --vh-radius-sm: 0.5rem;
    --vh-radius-md: 0.75rem;
    --vh-radius-lg: 1rem;
  }
  ```
- Replace ALL hardcoded Tailwind color values with `var(--vh-*)` references
- **Success criteria:** Default theme looks 100% identical after refactor

### Light Mode (Frontend)
- `data-theme="light"` / `data-theme="dark"` on `<html>`
- Light variable overrides:
  ```css
  [data-theme="light"] {
    --vh-bg-base: #f8fafc;
    --vh-bg-card: #ffffff;
    --vh-bg-elevated: #f1f5f9;
    --vh-border: #e2e8f0;
    --vh-text-primary: #0f172a;
    --vh-text-secondary: #1e293b;
    --vh-text-muted: #64748b;
    /* etc */
  }
  ```
- User preference: localStorage + account settings column
- Respects `prefers-color-scheme` as default
- Theme toggle in header (sun/moon icon)

### Light Mode Backend
- Add `theme_mode` column to users table (nullable, enum: dark/light/system)
- API endpoint to get/set user theme preference
- `forum_config` key for admin default (dark/light/user-choice)

## Week 2: Theme System

### Theme Backend
- Expand `theme.json` schema:
  ```json
  {
    "id": "my-theme",
    "name": "My Theme",
    "version": "1.0.0",
    "author": "Author Name",
    "description": "A clean theme",
    "homepage": "https://example.com",
    "tags": ["light", "minimal"],
    "mode": "dark|light|both",
    "preview": {
      "screenshot": "preview.png"
    }
  }
  ```
- Theme CRUD endpoints:
  - GET /api/admin/themes — list installed themes
  - POST /api/admin/themes — upload theme zip
  - PUT /api/admin/themes/{slug}/activate — set active theme
  - DELETE /api/admin/themes/{slug} — remove theme
  - GET /api/admin/themes/{slug}/preview — serve screenshot
  - POST /api/admin/themes/export — export current customizations as theme zip
- Theme storage: `storage/themes/{slug}/` (theme.json + theme.css + preview.png)
- Active theme CSS served via API endpoint or injected in page head

### Theme Admin UI (Frontend)
- Admin → Appearance → Themes page
- Grid of theme cards with:
  - Screenshot preview (or color swatch fallback)
  - Name, author, version, description, tags
  - Active indicator (checkmark badge)
  - Activate / Delete buttons
  - Preview mode (apply temporarily without saving)
- Upload theme button (drag & drop or file picker)
- "Export Current" button — packages current custom CSS as downloadable theme.zip

### Live CSS Editor (Frontend)
- Admin → Appearance → Custom Code (upgrade existing)
- Split pane: editor left, live preview right (iframe or real-time apply)
- CSS variable reference panel (collapsible sidebar listing all `--vh-*` vars with current values and color swatches)
- Syntax highlighting (CodeMirror or Monaco)
- "Export as Theme" button from editor

### Bundled Themes
Create 3 themes that ship with VoltexaHub:
1. **Default Dark** — current purple/dark theme (the default)
2. **Daylight** — clean light theme, blue accent
3. **Midnight** — deep navy/teal, different vibe from default

Each packaged as a proper theme.zip in `resources/themes/`.

## Theme Package Spec

```
my-theme.zip
├── theme.json       # required — manifest
├── theme.css        # required — variable overrides + custom styles
└── preview.png      # optional — 1200x675px screenshot
```

Minimal theme.css:
```css
:root {
  --vh-bg-base: #f8fafc;
  --vh-bg-card: #ffffff;
  --vh-accent: #6366f1;
  --vh-text-primary: #0f172a;
  --vh-border: #e2e8f0;
}
```
