# v0.7.1 Build Plan
**Target release:** ~2 weeks after v0.7.0 ships (est. April 5, 2026)
**Theme:** Community Themes + Polish

---

## 🎯 Primary Goal
Give the community the ability to build and share custom themes without touching Vue source code. The CSS custom properties refactor is the unlock for everything else.

---

## Sprint 1 — CSS Variables Refactor (Week 1)

### Backend (voltexahub/)
- No backend changes needed this sprint

### Frontend (voltexaforum/)
**CSS Custom Properties architecture**
- Audit every hardcoded color in the codebase (`bg-[#0a0a0f]`, `text-[#7c3aed]`, etc.)
- Define a complete set of CSS variables in a `:root` block in `app.css`:
  ```css
  :root {
    --vh-bg-base: #0a0a0f;
    --vh-bg-card: #0f0f1a;
    --vh-bg-elevated: #13131f;
    --vh-border: #1e1e30;
    --vh-border-subtle: #161622;
    --vh-accent: #7c3aed;
    --vh-accent-hover: #6d28d9;
    --vh-accent-soft: rgba(124,58,237,0.1);
    --vh-accent-light: #a78bfa;
    --vh-text-primary: #f1f5f9;
    --vh-text-secondary: #e2e8f0;
    --vh-text-muted: #94a3b8;
    --vh-text-faint: #475569;
    --vh-success: #10b981;
    --vh-warning: #f59e0b;
    --vh-danger: #ef4444;
    --vh-info: #3b82f6;
    --vh-radius-sm: 0.5rem;
    --vh-radius-md: 0.75rem;
    --vh-radius-lg: 1rem;
    --vh-font-base: 'Inter', system-ui, sans-serif;
    --vh-font-mono: 'JetBrains Mono', monospace;
  }
  ```
- Replace all hardcoded Tailwind color values with `var(--vh-*)` references
- Test: default theme looks identical after refactor

## Sprint 1 — Theme System v2 (Week 1 continued)

### Backend (voltexahub/)
**Expand theme.json schema**
```json
{
  "id": "my-theme",
  "name": "My Theme",
  "version": "1.0.0",
  "author": "Your Name",
  "description": "A clean light theme for VoltexaHub",
  "homepage": "https://example.com",
  "tags": ["light", "minimal"],
  "preview": {
    "bg": "#ffffff",
    "accent": "#6366f1",
    "screenshot": "preview.png"
  }
}
```
- Add `screenshot` support — theme zips can include a `preview.png`, served via `/api/themes/{slug}/preview`
- Add theme metadata endpoint: `GET /api/admin/themes/{slug}`

### Frontend (voltexaforum/)
**Theme admin UI overhaul**
- Replace gradient swatch cards with proper preview cards (screenshot if available, swatches fallback)
- Show author, version, description, tags on each card
- Active theme indicator (checkmark badge)
- Confirm dialog before switching themes
- "Preview" mode — apply theme temporarily without saving (session-only)

## Sprint 2 — Light Mode + Theme Authoring Docs (Week 2)

### Frontend (voltexaforum/)
**Light mode support**
- Add `data-theme="light"` / `data-theme="dark"` toggle on `<html>`
- Define light mode variable overrides:
  ```css
  [data-theme="light"] {
    --vh-bg-base: #f8fafc;
    --vh-bg-card: #ffffff;
    --vh-border: #e2e8f0;
    --vh-text-primary: #0f172a;
    /* etc */
  }
  ```
- User preference saved to localStorage + account settings
- Respects `prefers-color-scheme` as default if no preference set
- Admin can force a theme's mode (lock to dark, lock to light, or user choice)

**Theme authoring experience**
- Live CSS editor in admin → Appearance → Custom Code with real-time preview (iframe or apply-on-type with debounce)
- CSS variable reference panel in the editor (list all `--vh-*` vars with their current values)
- "Export as theme" button — takes current custom CSS and packages it as a downloadable theme.zip

### Backend (voltexahub/)
- `GET /api/themes/{slug}/preview` — serve screenshot image
- Store `theme_mode` preference in `forum_config` (dark/light/user)
- User settings: add `theme_mode` column to users table

## Theme Packaging Spec (for docs)
When v0.7.1 ships, publish a theme authoring guide at docs.voltexahub.com:

```
my-theme.zip
├── theme.json       # required — manifest
├── theme.css        # required — your variable overrides + custom styles
└── preview.png      # optional — 1200x675px screenshot
```

Minimal `theme.css` to change colors:
```css
:root {
  --vh-bg-base: #f8fafc;
  --vh-bg-card: #ffffff;
  --vh-accent: #6366f1;
  --vh-accent-light: #818cf8;
  --vh-text-primary: #0f172a;
  --vh-text-muted: #64748b;
  --vh-border: #e2e8f0;
}
```
That's all you need for a full color theme.
