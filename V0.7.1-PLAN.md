# v0.7.1 Build Plan
**Target release:** ~1 week (est. March 29, 2026)
**Theme:** Polish, Security Hardening, QoL

---

## Quick Wins

### Skeleton Loaders (Frontend)
- Replace spinner/loading states with skeleton placeholders on:
  - Thread list (home page + forum view)
  - Profile page
  - Awards catalog
- Use animated pulse shimmer (Tailwind `animate-pulse` on gray blocks)

### DB Automated Backups (Backend)
- Daily `mysqldump` via Laravel scheduled command
- Store last 7 days in `storage/backups/`, auto-prune older
- Admin dashboard widget showing last backup time + file size
- Manual backup trigger from admin panel (reuse existing export logic)

### Thread Activity Indicator (Frontend + Backend)
- "X new replies since your last visit" badge on thread list items
- Track per-user last-read timestamp per thread
- Backend: `thread_reads` table (user_id, thread_id, last_read_at)
- Frontend: subtle badge/pill on thread row

## Security Hardening (carryover from v0.7.0)

### Admin Re-Auth (Backend + Frontend)
- Password confirm (or MFA code) before destructive admin actions:
  - Delete user, restore database, reset MFA, change admin roles
- Backend middleware `AdminReAuth` — checks recent auth timestamp
- Frontend modal prompt when action requires re-auth

### Inactive Session Auto-Cleanup (Backend)
- Scheduled command to purge Sanctum tokens unused for 30+ days
- Run daily via Laravel scheduler
- Log count of purged sessions

### IP-Level Brute Force Protection (Backend)
- Track failed login attempts per IP (not just per account)
- Block IP for 1 hour after 20 failed attempts across any accounts
- Use cache (Redis) for tracking
- Admin can view/unblock IPs from security settings
